# 테스트 전략

## 개요

- 복잡한 로직이 많이 사용되어야하므로 테스트를 충실히 수행한다.
- json <-> html UI, json <-> sql  이런 동작은 서버 python으로도 수행되며 javascript로도 수행된다.

## tools

- python : pytest
- javascript : jest, playwright

## test folder

- python : tests
- javascript : tests_js

## folder 구성 예

```text
auto-board/
├── tests/              # Python 테스트
│   ├── conftest.py
│   ├── test_boards.py
│   ├── test_metadata.py
│   └── test_api/
│       └── test_board_api.py
├── tests_js/           # JavaScript 테스트
│   ├── unit/
│   │   ├── utils/
│   │   │   └── validation.test.js
│   │   └── components/
│   │       └── field-renderer.test.js
│   ├── integration/
│   │   └── alpine/
│   │       └── board-form.test.js
│   └── e2e/
│       ├── board-crud.spec.js
│       └── record-crud.spec.js
├── pytest.ini
├── vitest.config.js
└── playwright.config.js
```

## 테스트 코드 작성 방법

- 아주 간단한 코드부터 테스트한다.
